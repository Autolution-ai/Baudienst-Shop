## 1. Pre-Writing Checklist

**When to use:** Before any word of copy is written. If you can't complete this checklist, you don't have enough upstream thinking to write the page yet. Go research more, don't open the doc.

- [ ] Schwartz awareness stage diagnosed (1–5) — write the number down: \_\_\_
- [ ] Market sophistication diagnosed (1–5) — write the number down: \_\_\_
- [ ] Dunford 5+1 components defined:
  - [ ] Competitive alternatives (what would they use instead?)
  - [ ] Unique attributes (what only we have)
  - [ ] Value (what those attributes enable)
  - [ ] Who cares (best-fit customers)
  - [ ] Category (where we belong)
  - [ ] Trends (relevant market trends we ride)
- [ ] Persona identified — written as role × life-stage × awareness-stage
- [ ] VoC research complete:
  - [ ] 10+ reviews mined (own + competitor)
  - [ ] 3+ customer interviews if possible (Mom Test compliant)
  - [ ] Support tickets / sales call transcripts scanned
  - [ ] Reddit / forum / community search done
- [ ] Voice Constraint Card built (10 always-use words, 10 never-use, signature moves, anti-moves) — see [voice-tone.md](voice-tone.md)
- [ ] Big Idea drafted in ONE sentence; passes Ford's 4 tests:
  - [ ] Is it new?
  - [ ] Is it big?
  - [ ] Does it solve a real problem?
  - [ ] Is it provable?
- [ ] Framework selected and matches awareness × sophistication:
  - [ ] Stage 1–2 (Unaware/Problem-aware) → Story / PAS / PASTOR lead
  - [ ] Stage 3 (Solution-aware) → Mechanism-led / "Hidden reason" lead
  - [ ] Stage 4 (Product-aware) → Offer / Proof / Specificity lead
  - [ ] Stage 5 (Most aware) → Identity / Reminder / Reorder lead
- [ ] 20 objections predicted; each mapped to a specific page element (headline, sub, proof block, FAQ, CTA caption) — see [objection-prediction.md](objection-prediction.md)
- [ ] Risk reversal defined (guarantee, free trial, money-back, performance-based, etc.)
- [ ] CTA defined — single primary action (no multi-CTA confusion)
- [ ] Channel-specific length / format established (word count, asset specs, deliverable list)
- [ ] Mandatory inclusions documented (legal claims, proof points, brand-required phrases)
- [ ] Success metric defined (conversion target, response rate, CPL, demo bookings)

## 2. Edit-Pass Checklist

**When to use:** After the first draft. The first draft is for getting it out. This pass is for making it work. Budget 2× the writing time for this pass.

### Voice & rhythm

- [ ] Read aloud, top to bottom — every sentence flows in real speech
- [ ] Cut 30–50% on second pass (Schwartz: "the second draft is the first one, minus")
- [ ] Sentence-length distribution varied (Provost cadence: short. Medium. Long.)
- [ ] Active voice, 2nd person ("you"), present tense as defaults
- [ ] "You" beats "we" 3:1 or better — count them

### Specificity

- [ ] Replace every generic adjective with a number (not "fast" → "ships in 4 days")
- [ ] Replace every "many / lots / a ton" with the exact count
- [ ] Replace every abstraction with sensory specifics (not "save time" → "leave the office at 5:30 instead of 7")
- [ ] No unsupported superlatives ("best / world-class / leading" without proof)
- [ ] Cliché count = 0 (game-changer, revolutionary, seamless, cutting-edge, robust)

### Headline & hook

- [ ] Headline passes the 4 Us:
  - [ ] Useful
  - [ ] Urgent
  - [ ] Unique
  - [ ] Ultra-specific
- [ ] First sentence passes Sugarman test — makes you read the second
- [ ] Headline tested against the 4 buyer concerns (Will it work / for me / how fast / how risky)

### AI tells (quick scan — see full list in Section 4)

- [ ] No em-dash overuse (count em-dashes; cut 50%)
- [ ] No three-item-list reflex (force two- or four-item where possible)
- [ ] No banned words: delve, tapestry, leverage, crucial, pivotal, vibrant, nestled, boasts, garner, intricate, interplay, landscape, meticulous, testament, underscore, enduring

### The "human signature"

- [ ] One contrarian point AI couldn't have written
- [ ] One made-from-life specific detail (a real story, a real number, a real moment)
- [ ] Big Idea visible in headline AND reinforced at 3+ points throughout

### Structure

- [ ] Proof placed near each major claim — not buried at the bottom
- [ ] Proof placed above the CTA, not after
- [ ] FAQ pre-empts the top 8 objections (not the 8 you wish you got)
- [ ] CTA caption removes the last objection ("No credit card required" / "Cancel any time" / "Free 14-day trial")

## 3. Page-Audit Diagnostic

**When to use:** After Edit-Pass. Score each buyer-concern out of 10 by reading the page as a skeptical buyer. The lowest score is what's killing conversion.

- [ ] **Will it actually work?** Score: \_\_\_ / 10
  - Evidence shown? Logos, case studies, numbers, demos, third-party validation?
- [ ] **Will it work for ME specifically?** Score: \_\_\_ / 10
  - Segmented case studies, persona-mirroring language, role-specific examples?
- [ ] **How fast will I see results?** Score: \_\_\_ / 10
  - Time-to-value stated? Day-by-day breakdown? Quick-win promise?
- [ ] **How risky is this purchase?** Score: \_\_\_ / 10
  - Guarantee? Free trial? Transparent terms? Cancel/refund visible? Reverse-risk statement?

**Rule:** Lowest score = fix next. Any score below 6 = killing conversion. Don't ship a page with any score under 6.

**Bonus diagnostic — read top of page only, then look away:**

- [ ] Can you state the offer in one sentence after reading just the above-fold?
- [ ] Can you state the target customer?
- [ ] Can you state the primary outcome?

If no to any → above-fold is failing.

## 4. AI-Slop Scan Checklist

**When to use:** After Edit-Pass, before publish. AI-generated copy has a fingerprint; this list strips it. Even fully human drafts will trip several of these — that's normal; clean them up. See [ai-workflow.md](ai-workflow.md) for the deeper teardown.

### Vocabulary tells (banned-words scan)

Scan and replace every instance of:

- [ ] delve
- [ ] tapestry
- [ ] leverage
- [ ] crucial
- [ ] pivotal
- [ ] vibrant
- [ ] nestled
- [ ] boasts
- [ ] garner
- [ ] intricate
- [ ] interplay
- [ ] landscape
- [ ] meticulous
- [ ] testament
- [ ] underscore
- [ ] enduring
- [ ] align with
- [ ] foster
- [ ] highlight (as verb)
- [ ] showcase (as verb)
- [ ] robust
- [ ] seamless
- [ ] holistic
- [ ] streamline

### Structural tells

- [ ] No "It's not X — it's Y" parallelism (default AI rhetorical move)
- [ ] No three-item lists everywhere (force two- or four-item where it reads better)
- [ ] No "in conclusion / ultimately / moreover / however / furthermore / additionally"
- [ ] No "Let's break it down" / "Here's the thing" / "Great question!"
- [ ] No "In today's [adjective] landscape" / "in an ever-evolving"
- [ ] No hedging-then-contradicting ("Though relatively unknown, it represents…")
- [ ] No mechanical section-heading bolding (e.g. "**Key benefit:** …" pattern repeated)

### Punctuation tells

- [ ] Em-dash count reduced by 50%+ from first draft
- [ ] No curly quotes from copy-paste artifacts (’ ‘ “ ” mixed with straight)
- [ ] No mechanical bolding for emphasis (every paragraph has a bolded phrase = AI tell)
- [ ] No title-case headings where sentence-case is brand standard

### Grammar tells

- [ ] No "serves as / stands as / marks the" — replaced with "is"
- [ ] No "boasts / features / maintains" — replaced with "has"
- [ ] No present participles tacked on ("…ing the way for new approaches")
- [ ] No empty modifiers ("truly / really / very / simply")

### Anti-slop edit pass (8 steps, in order)

Run these in sequence — each builds on the last.

1. [ ] Replace every adjective with a number
2. [ ] Replace every abstraction with sensory specifics
3. [ ] Cut em-dashes 50%
4. [ ] Cut "in conclusion / ultimately / moreover / however"
5. [ ] Break the rule-of-three reflex (find every three-item list; convert at least half)
6. [ ] Find one contrarian point; amplify it
7. [ ] Add one made-from-life detail AI couldn't have invented
8. [ ] Read aloud — if anything sounds like a press release, rewrite it

## 5. Pre-Publish Objection Audit

**When to use:** Just before publishing. Cross-reference with [objection-prediction.md](objection-prediction.md). If you can't tick every box, the page leaks conversion.

- [ ] All 8 universal objections addressed somewhere on the page:
  - [ ] Too expensive
  - [ ] Not enough time
  - [ ] Won't work for my situation
  - [ ] Don't trust you
  - [ ] Need to think about it
  - [ ] Need to ask someone else
  - [ ] Tried something similar; didn't work
  - [ ] Not the right time
- [ ] Role-specific objections for the primary persona addressed (e.g. CFO wants ROI; CTO wants security; End-user wants ease)
- [ ] Awareness-stage objections addressed:
  - [ ] Stage 2 (Problem-aware) → problem validation present
  - [ ] Stage 3 (Solution-aware) → why-this-mechanism explained
  - [ ] Stage 4 (Product-aware) → offer specificity + comparison
  - [ ] Stage 5 (Most aware) → identity reinforcement + reorder ease
- [ ] Top 3 "embarrassing" objections pre-empted (the ones the buyer won't say out loud)
- [ ] Each VoC-discovered objection mapped to a specific page element
- [ ] 4 Buyer Concerns (Section 3) each score ≥ 7
- [ ] Risk reversal explicit and visible above the CTA — not in the footer
- [ ] FAQ contains the toughest objection, not the convenient one
- [ ] Negative reviews / criticism handled (acknowledge → reframe → resolve)
- [ ] Competitor comparison present where market sophistication ≥ Stage 4

## 6. Voice Consistency Checklist

**When to use:** On every published piece, every channel. The voice card is the source of truth. See [voice-tone.md](voice-tone.md).

- [ ] Matches Voice Constraint Card (10 always-use words present; 10 never-use words absent)
- [ ] Tone position on NN/g 4 axes (Funny↔Serious / Formal↔Casual / Respectful↔Irreverent / Enthusiastic↔Matter-of-fact) consistent with brand
- [ ] Reading level matches audience target (Hemingway score, Flesch-Kincaid grade as set in brand guidelines)
- [ ] Rhythm varies (Provost cadence — short. medium. longer sentences that build.)
- [ ] Active voice, 2nd person, present tense as defaults
- [ ] "You" beats "we" 3:1 or better
- [ ] Sentence-length distribution varied (not all 12-word sentences; not all 28-word sentences)
- [ ] No try-hard irreverence ("oof / yikes / spicy take" when brand voice isn't this)
- [ ] No corporate-speak ("solutions / synergies / leverage / stakeholders") unless brand voice IS corporate
- [ ] Founder / personality presence consistent (where brand depends on founder voice)
- [ ] Signature moves present (the things only this brand says)
- [ ] Anti-moves absent (the things this brand never says)

## 7. Channel-Specific QA Checklists

**When to use:** Per channel. Each channel has different physics. Run the right sub-list. See [channels.md](channels.md) for full conventions.

### Landing page

- [ ] Above-fold complete (headline, sub, primary CTA, one proof signal)
- [ ] Single primary CTA — no decision paralysis
- [ ] Social proof present (logos, testimonials, ratings, counts)
- [ ] FAQ pre-empts top 8 objections
- [ ] Mobile-tested at 375px width
- [ ] No CLS on hero (image dimensions set)
- [ ] CTA caption removes the last objection

### Long-form sales page / VSL script

- [ ] Lead type matches awareness stage (story / problem / mechanism / offer / proof)
- [ ] Mechanism revealed (the unique reason this works) — by minute 10 in a VSL, by 1/3 down a sales page
- [ ] Proof density appropriate to claim size (bigger claim → more proof)
- [ ] Offer stack visible (everything they get, each line with implied value)
- [ ] Urgency is genuine (real deadline, real scarcity, real cohort cap — not fabricated)
- [ ] Risk reversal stronger than the price ($X guarantee on $Y offer where X ≥ Y)
- [ ] Multiple CTAs throughout, all the same action

### Email

- [ ] Subject line < 50 characters (mobile preview)
- [ ] Preheader present and adds info — not a duplicate of subject
- [ ] Single CTA per email
- [ ] Mobile-tested (60%+ of opens)
- [ ] Unsubscribe + physical address compliant (CAN-SPAM / GDPR)
- [ ] Plain-text version reads cleanly
- [ ] No image-only emails (clipping, deliverability)
- [ ] First line is NOT "Hope you're well" — uses Sugarman slippery slide

### Cold outreach

- [ ] < 100 words total
- [ ] Single ask
- [ ] Specific trigger event referenced (recent news, hire, launch, role change)
- [ ] No praise ("love your work / huge fan") — buyers smell it
- [ ] No generic "I noticed you…" / "Hope this email finds you well"
- [ ] Specific to recipient (a 2-second skim proves it isn't a blast)
- [ ] CTA is low-friction ("worth a 15-min look?" beats "let's schedule a call")
- [ ] P.S. used as second hook if needed

### LinkedIn post

- [ ] Hook in first line (before the "see more" fold)
- [ ] Line breaks / whitespace (not a wall of text)
- [ ] Narrative arc (setup → tension → payoff) — not bullet dump
- [ ] Comment-driving CTA ("what would you add?" / "agree?")
- [ ] Not broetry-without-insight (short lines must earn their break)
- [ ] No hashtag spam (3 max, in body)
- [ ] No "I'm humbled / honored to announce" (LinkedIn cliché)

### Twitter / X thread

- [ ] Hook tweet uses near-full 280 character budget for context
- [ ] Progression — each tweet earns the next read
- [ ] Payoff in final tweet (lesson, framework, conclusion)
- [ ] Optional bookmark/follow CTA at end
- [ ] No "1/" numbering reflex if it weakens the hook
- [ ] No "🧵" emoji if it telegraphs corporate hand

### TikTok / Reels / Shorts

- [ ] 3-second hook (visual + verbal)
- [ ] On-screen captions (85% of views are muted)
- [ ] Talking-head or B-roll matches the message
- [ ] No slow-open / no logo intro
- [ ] Pattern interrupt within first 5 seconds
- [ ] Loop-ability if applicable (ends where it began)
- [ ] CTA in caption (in-video CTAs cap reach)

### Meta ad (Facebook / Instagram)

- [ ] Hook in primary text first line (above "more" fold)
- [ ] Single image or pattern interrupt as the visual
- [ ] < 125 character primary text in mobile preview before truncation
- [ ] Headline (under image) earns its 27 characters
- [ ] CTA button matches offer ("Learn more" for awareness; "Shop now" for direct)
- [ ] Mobile preview tested (where 95% of impressions live)
- [ ] No stock-photo-aesthetic (kills scroll-stop rate)

### VSL (video sales letter)

- [ ] Snap suggestion (Cialdini pre-suasion) in first 5 seconds — primes the entire watch
- [ ] Mechanism revealed by minute 10 (or viewers drop)
- [ ] Proof stack delivered before the offer
- [ ] Urgency is real (cohort close, price increase, capacity cap)
- [ ] Order button visible at and after minute 30 for long-form VSLs
- [ ] Auto-play with sound muted as default — captions present
- [ ] Speed-up resistance built in (must-watch moments seeded throughout)

## 8. Ethics Check

**When to use:** Before every publish, no exceptions. Ethical copy isn't a niceness tax — it's the lowest-refund, highest-LTV path.

- [ ] Is every claim defensible with evidence? (Could you back it up in court / on a podcast / in front of a customer?)
- [ ] Is the scarcity / urgency REAL (not fabricated countdown timers that reset)?
- [ ] Does the guarantee actually pay out? (Has the team committed to honoring it?)
- [ ] Would the founder say this to a customer's face?
- [ ] Are we connecting genuine value to genuine need — or fabricating need?
- [ ] Would high refund rates expose us? (If yes → the copy outran the product)
- [ ] Is personalization genuine or surveillance-coded? ("We saw you visited 3 times" reads creepy, not helpful)
- [ ] No dark patterns (forced continuity, hidden recurring charges, fake reviews, fake social proof)
- [ ] No false urgency ("only 2 left" when inventory is unlimited)
- [ ] No "harvested" testimonials (used without permission or out of context)
- [ ] Legal review complete for regulated claims (health, finance, income, weight-loss)
- [ ] Accessibility considered (no copy that requires color, motion, or audio to understand)

## 9. Final "Send It?" Check

**When to use:** 30 seconds before publish. Last skim. If anything trips, fix and re-skim.

- [ ] Headline still makes me want to read on
- [ ] First sentence makes me want to read the second
- [ ] Proof is in the right place (near the claim, above the CTA)
- [ ] CTA is unambiguous (one action, one outcome)
- [ ] No typos in CTAs, headlines, or named people / companies
- [ ] All hyperlinks work (test the primary CTA URL especially)
- [ ] Mobile view tested at 375px
- [ ] If brand has a voice card — this matches it
- [ ] If shipped to email — sent to self, mobile preview verified
- [ ] If shipped to web — published in incognito, full page rendered, CTA clicked
- [ ] If shipped as ad — preview in placement (Facebook feed mobile, IG story, TikTok native)
- [ ] Tracking / analytics in place (UTM, event, conversion pixel) — measurement is part of the work
- [ ] Author's name / contact attached where relevant
- [ ] Internal team notified (sales, support, success) so they can answer questions the copy generates

## Meta-Checklist: Did You Run the Right Checklist?

- [ ] Phase 0 (Brief discovery) → No checklist; use the Creative Brief Template in [workflows.md](workflows.md)
- [ ] Phase 1–4 (Position, audience, objection, big idea) → **Pre-Writing Checklist** (Section 1)
- [ ] Phase 5–6 (Outline, draft) → No checklist; write
- [ ] Phase 7 (Edit) → **Edit-Pass** (2) → **AI-Slop Scan** (4) → **Voice Consistency** (6)
- [ ] Phase 8 (Audit + test) → **Page-Audit Diagnostic** (3) → **Pre-Publish Objection Audit** (5) → **Channel-Specific QA** (7) → **Ethics Check** (8) → **Final "Send It?"** (9)

If you ran all nine in order, you ran the discipline. Most underperforming copy skipped 3–5 of these. The discipline is the moat.
